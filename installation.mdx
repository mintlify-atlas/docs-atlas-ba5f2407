## Core Dependencies

Tune Me In includes these key dependencies:

### Hydrogen & Shopify

```json package.json
{
  "dependencies": {
    "@shopify/hydrogen": "0.7.0"
  }
}
```

The Shopify Hydrogen framework provides:
- Server-side rendering with React 18
- Streaming SSR for fast page loads
- Built-in commerce components
- Storefront API integration

### Sanity Integration


## Configuration

<Tabs>
  <Tab title="Using Demo Data">
    ### Run with Demo Data
    
    The starter comes pre-configured with demo data. You can start developing immediately:
    
    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>
    
    Visit `http://localhost:3000` to see the storefront with demo products.
    
    <Info>
      The demo configuration points to:
      - **Shopify Store:** `sanity-dev-store.myshopify.com`
      - **Sanity Project:** `wfr1r0dw` (dataset: `production`)
    </Info>
  </Tab>
  
  <Tab title="Using Your Own Data">
    ### Configure Your Own Store
    
    To use your own Shopify store and Sanity project, update both configuration files.
    
    <Steps>
      <Step title="Get Shopify Credentials">
        1. Log in to your Shopify admin
        2. Go to **Settings** → **Apps and sales channels**
        3. Create a custom app or use an existing one
        4. Enable **Storefront API** access
        5. Copy your **Storefront Access Token** and **Store Domain**
      </Step>
      
      <Step title="Update shopify.config.js">
        Edit `shopify.config.js` at the root of your project:
        
        ```javascript shopify.config.js
        import sanityConfig from './sanity.config';

        export default {
          graphqlApiVersion: 'unstable',
          locale: 'en-us',
          storeDomain: 'your-store.myshopify.com', // Your store domain
          storefrontToken: 'your-storefront-access-token', // Your token
          sanity: sanityConfig,
        };
        ```
        
        <Warning>
          Keep your `storefrontToken` secure. Consider using environment variables for production:
          ```javascript
          storefrontToken: import.meta.env.VITE_SHOPIFY_STOREFRONT_TOKEN
          ```
        </Warning>
      </Step>
      
      <Step title="Get Sanity Credentials">
        1. Log in to your [Sanity account](https://www.sanity.io/manage)
        2. Find or create a project
        3. Note your **Project ID** (visible in project settings)
        4. Choose your **Dataset** name (usually `production`)
      </Step>
      
      <Step title="Update sanity.config.js">
        Edit `sanity.config.js` at the root of your project:
        
        ```javascript sanity.config.js
        export default {
          apiVersion: 'v2021-06-07',
          dataset: 'production', // Your dataset name
          projectId: 'your-project-id', // Your Sanity project ID
          useCdn: true, // Use CDN for faster queries
        };
        ```
        
        <Tip>
          Set `useCdn: false` during development if you want to see content changes immediately without CDN caching.
        </Tip>
      </Step>
      
      <Step title="Set Up Sanity Connect">
        To sync your Shopify products to Sanity, install Sanity Connect:
        
        1. Install the [Sanity Shopify Studio](https://github.com/sanity-io/sanity-shopify-studio)
        2. Configure [Sanity Connect for Shopify](https://www.sanity.io/docs/sanity-connect-for-shopify)
        3. Run an initial sync to import your products
        
        <Info>
          Sanity Connect automatically syncs your products, keeping your Sanity dataset up to date with Shopify changes.
        </Info>
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Environment Variables (Optional)

For better security in production, use environment variables:

<Steps>
  <Step title="Create .env File">
    Create a `.env` file at the root of your project:
    
    ```bash .env
    VITE_SHOPIFY_STORE_DOMAIN=your-store.myshopify.com
    VITE_SHOPIFY_STOREFRONT_TOKEN=your-token
    VITE_SANITY_PROJECT_ID=your-project-id
    VITE_SANITY_DATASET=production
    ```
  </Step>
  
  <Step title="Update Configuration Files">
    Reference environment variables in your config:
    
    ```javascript shopify.config.js
    export default {
      storeDomain: import.meta.env.VITE_SHOPIFY_STORE_DOMAIN || 'sanity-dev-store.myshopify.com',
      storefrontToken: import.meta.env.VITE_SHOPIFY_STOREFRONT_TOKEN || 'default-token',
      // ...
    };
    ```
    
    ```javascript sanity.config.js
    export default {
      projectId: import.meta.env.VITE_SANITY_PROJECT_ID || 'wfr1r0dw',
      dataset: import.meta.env.VITE_SANITY_DATASET || 'production',
      // ...
    };
    ```
  </Step>
  
  <Step title="Add to .gitignore">
    Make sure `.env` is in your `.gitignore`:
    
    ```bash .gitignore
    .env
    .env.local
    .env.production
    ```
  </Step>
</Steps>

<Warning>
  Never commit sensitive credentials to version control. Always use environment variables for production deployments.
</Warning>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Node version error">
    Tune Me In requires Node.js 14.0 or higher. Update Node.js:
    
    ```bash
    # Using nvm
    nvm install 14
    nvm use 14
    
    # Or download from nodejs.org
    ```
  </Accordion>
  
  <Accordion title="Installation fails">
    Try these steps:
    
    1. Clear npm cache:
       ```bash
       npm cache clean --force
       ```
    
    2. Delete node_modules and lock file:
       ```bash
       rm -rf node_modules package-lock.json
       npm install
       ```
    
    3. Use a different package manager (try yarn or pnpm)
  </Accordion>
  
  <Accordion title="Shopify connection error">
    Verify your Shopify credentials:
    
    - Store domain should be in format: `your-store.myshopify.com`
    - Storefront token must have Storefront API access enabled
    - Check token hasn't expired
    
    Test your credentials using the Shopify Storefront API explorer.
  </Accordion>
  
  <Accordion title="Sanity data not loading">
    Check your Sanity configuration:
    
    - Project ID is correct (visible in project settings)
    - Dataset exists and has data
    - Products are synced from Shopify using Sanity Connect
    - API permissions allow read access
    
    Test your connection:
    ```bash
    npx @sanity/cli dataset list --project=your-project-id
    ```
  </Accordion>
  
  <Accordion title="Build errors">
    Common build issues:
    
    - Ensure all dependencies are installed: `npm install`
    - Check Node.js version: `node --version`
    - Clear Vite cache: `rm -rf node_modules/.vite`
    - Update dependencies: `npm update`
  </Accordion>
</AccordionGroup>
